# Technical Documentation: MiniLang Compiler & Virtual Machine Architecture

This document provides the comprehensive architectural specification, grammar definitions, and runtime execution mechanics for **MiniLang**—a compiled, strongly typed (integer-only), block-scoped language executed via a custom Stack-Machine Virtual Machine (VM).

---

## 1. Language Specification & EBNF Grammar

MiniLang is designed to demonstrate high-level structured control flows (`if-else`, `while` loops), modular scoped routines (functions), and global/local memory state separation.

### Grammar Formalization (EBNF)

```ebnf
Program     ::= ( GlobalDecl | FuncDecl )*
GlobalDecl  ::= "var" Identifier ";"
FuncDecl    ::= "def" Identifier "(" ")" "{" LocalDecl* Stmt* "}"
LocalDecl   ::= "var" Identifier ";"
Stmt        ::= AssignStmt | IfStmt | WhileStmt | PrintStmt | ReturnStmt | BlockStmt
AssignStmt  ::= Identifier "=" Expr ";"
IfStmt      ::= "if" "(" Expr ")" Stmt ( "else" Stmt )?
WhileStmt   ::= "while" "(" Expr ")" Stmt
PrintStmt   ::= "print" Expr ";"
ReturnStmt  ::= "return" Expr ";"
BlockStmt   ::= "{" Stmt* "}"
Expr        ::= Term ( ( "+" | "-" | "==" | ">" | "<" ) Term )*
Term        ::= Factor ( ( "*" | "/" ) Factor )*
Factor      ::= Number | Identifier | Identifier "(" ")" | "(" Expr ")"

```

## 2. System Pipeline Architecture

The MiniLang execution framework consists of a single-pass hybrid AOT (Ahead-of-Time) compiler combined with a physical emulation runtime layer.

### Pipeline Stages

1. **Lexical Analysis (The Lexer):** Strips comments and whitespace, converting raw strings into a continuous token sequence (`Token(TOK_IF)`, `Token(TOK_NUM, 5)`, etc.).
2. **Syntax Analysis & Single-Pass Compilation:** Evaluates grammatical validity using a recursive-descent strategy. It registers variable definitions into explicit symbol tables while simultaneously producing target code.
3. **Backpatching Label Resolution:** Manages conditional statements and loop boundaries. The compiler emits structural code with blank destination pointers, caching their positions to retrospectively patch the concrete destination indices once they are parsed.
4. **Virtual Machine Interpretation Loop:** Accepts raw generated bytecode tuples and passes them through an internal Fetch-Decode-Execute CPU emulator cycle.

## 3. Scope & Memory Management Topology

MiniLang implements an optimized memory model that completely avoids heap allocations and garbage collection overhead.

| Memory Context | Allocation Mechanism | Scope Visibility | Addressing Mode |
| --- | --- | --- | --- |
| **Global Storage** | Evaluated once at start, stored in a persistent global lookup map. | Global across all functions. | Symbolic Named Lookup (`STORE_GLOBAL`, `LOAD_GLOBAL`) |
| **Local Frame** | Allocated sequentially within the local activation context via `ALLOC`. | Restricted to the parent function scope block. | Base-Relative Index Offset (`STORE_LOCAL`, `LOAD_LOCAL`) |

### Local Variable Frame Offsets

Local variables inside a function are assigned sequential integers based on their declaration order. For example:

```c
def sample() {
    var a; // Allocated index 0
    var b; // Allocated index 1
    a = 10; // Emits: STORE_LOCAL 0
}

```

## 4. Virtual Machine Runtime Mechanics

The execution engine replicates physical computer processor behavior using an instruction pointer (`IP`) and two isolated structural storage regions: **The Data Stack** and **The Call Stack**.

### A. The Data Stack

An internal, dynamic stack framework utilized to calculate expressions. For example, evaluating `result = (3 * 5) + 2` progresses as follows:

1. `PUSH 3` -> Stack: `[3]`
2. `PUSH 5` -> Stack: `[3, 5]`
3. `MUL`    -> Pops `5` and `3`, multiplies them, pushes result. Stack: `[15]`
4. `PUSH 2` -> Stack: `[15, 2]`
5. `ADD`    -> Pops `2` and `15`, adds them, pushes result. Stack: `[17]`
6. `STORE_LOCAL 0` -> Pops `17` and saves it into local index `0`. Stack: `[]`

### B. The Call Stack & Function Interactivity

When a function call is triggered (`CALL` instruction), MiniLang isolates the active execution context by pushing an explicit **Activation Record** onto the call stack.

* **Caller Interactivity:** The active instruction pointer position (`IP + 1`) and the current parent context's `locals` environment array are bundled together and pushed onto the Call Stack.
* **Frame Initialization:** The current execution frame clears its local tracking state array. The VM jumps to the requested function address, allocating fresh spaces using `ALLOC`.
* **The Return Handshake (`RET`):** The value at the top of the Data Stack is popped off to serve as the return payload. The VM pops the previous frame's saved layout state and instruction pointer location back into memory, then pushes the return value back onto the caller's active Data Stack.

## 5. Virtual ISA Instruction Reference

The following table serves as the definitive functional specification for MiniLang's target bytecode instructions:

| Opcode | Argument | Description | Stack Transformation |
| --- | --- | --- | --- |
| `PUSH` | `Value` | Pushes an explicit literal integer constant onto the Data Stack. | `[...] -> [..., Value]` |
| `ALLOC` | `Count` | Reserves `Count` slots in the current local frame array. | No Change |
| `LOAD_LOCAL` | `Index` | Reads the local array variable at `Index` and pushes it onto the Data Stack. | `[...] -> [..., LocalValue]` |
| `STORE_LOCAL` | `Index` | Pops the top value off the Data Stack and stores it into the local array at `Index`. | `[..., Value] -> [...]` |
| `LOAD_GLOBAL` | `Name` | Reads the global map variable associated with `Name` and pushes it onto the Data Stack. | `[...] -> [..., GlobalValue]` |
| `STORE_GLOBAL` | `Name` | Pops the top value off the Data Stack and assigns it to the global variable matching `Name`. | `[..., Value] -> [...]` |
| `ADD` / `SUB` | None | Pops the top two items (`b` then `a`), performs the operation, and pushes the result. | `[..., a, b] -> [..., a + b / a - b]` |
| `MUL` / `DIV` | None | Pops the top two items (`b` then `a`), performs the operation, and pushes the result. | `[..., a, b] -> [..., a * b / a / b]` |
| `CMPEQ` | None | Evaluates structural equivalence. Pushes `1` if the values match, otherwise `0`. | `[..., a, b] -> [..., 1 or 0]` |
| `CMPGT` / `CMPLT` | None | Evaluates relational inequality (`a > b` or `a < b`). Pushes `1` or `0`. | `[..., a, b] -> [..., 1 or 0]` |
| `JMP` | `Address` | Unconditionally shifts the Instruction Pointer (`IP`) to the target `Address`. | No Change |
| `JZ` | `Address` | Pops the top value. If it evaluates to `0` (False), shifts `IP` to the target `Address`. | `[..., Condition] -> [...]` |
| `CALL` | `Name` | Pushes the next instruction index and local environment onto the Call Stack, then branches to `Name`. | No Change |
| `RET` | None | Restores the previous parent execution environment from the Call Stack and returns execution control. | Handled via Return Handshake |
| `PRINT` | None | Pops the top value off the Data Stack and prints it to stdout. | `[..., Value] -> [...]` |
